## 2026-05-14 hardening addendum

This pass closed the feasible security / data-integrity / UX items that remained after the 2026-05-13 closure ledger, and converted larger schema/product-policy work into explicit backlog rows.

### Closed in code

- Mock section completion now requires canonical submitted Reading/Listening attempt evidence (`contentAttemptId`) and derives raw/scaled scores server-side, so forged client score fields are overwritten at completion time as well as report aggregation time.
- Live-room transitions now reject terminal bookings, validate transition metadata before persistence, restrict learner completion to bookings with finalized recording evidence, and return staff projections to assigned experts/admins.
- Chunked admin upload session mutation is scoped to the creating admin (`AdminUserId`) for upload-part, complete, and abort operations; cross-admin mutation attempts now fail as not found/no-op.
- Quality analytics routes moved off broad content-read into `AdminQualityAnalytics`: quality dashboard, Reading/Writing/Listening analytics, Listening attempt export, mock item-analysis, mock analytics, and mock risk list.
- Analytics ingestion now ignores oversized request bodies, caps property count/key/value sizes, redacts sensitive keys, drops nested objects, and persists only bounded primitive/array payloads.
- Audit log search/filter controls stay mounted on empty filtered results; mock leak-report filters only show counts for the currently loaded status, avoiding misleading inactive `(0)` chips.
- Diagnostic mock documentation/commentary now reflects the real loading contract: Suspense covers client-side router suspension, while the explicit loading branch owns the first API round-trip skeleton.
