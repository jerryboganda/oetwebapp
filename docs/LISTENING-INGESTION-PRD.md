# Listening Ingestion & End-to-End Compatibility — PRD

> Generated by ultrawork agent on 2026-05-05 from real sample analysis.
> Source data: `Project Real Content/Listening ( IMPORTANT NOTE = Same for All Professions )/Listening Sample {1,2,3}/`
> Each sample = `Question-Paper.pdf` + `Audio-Script.pdf` + `Answer-Key.pdf` + `Audio N/Audio N.mp3` (33–53 MB).
> Sample 2's `Listening Sample 2 Answers.pdf` is a **scanned image PDF** (no embedded text) — requires OCR.

## 1. Goal

Make every byte of the real Listening sample data **end-to-end compatible** with the existing platform:

1. Admin can upload the 3-PDF + MP3 quartet through the existing CMS.
2. AI extraction produces an editable 42-item draft (24 short-answer + 6 MCQ + 12 MCQ).
3. Admin reviews / edits / publishes via existing `ListeningStructureEditor` + `ListeningExtractMetadataEditor`.
4. Subscribed learner can play, answer, submit, see policy-safe review.
5. Unsubscribed learner sees the existing `ContentLockedNotice` upsell on every gated entry point (not just attempt-start).
6. All AI calls remain grounded; all admin AI is platform-only; full `AiUsageRecord` audit.

- Publish gate stays at canonical 24/6/12 = 42.

## 6. Acceptance criteria

1. Add UglyToad.PdfPig 0.1.10+ NuGet to `OetLearner.Api`. New `PdfPigPdfTextExtractor : IPdfTextExtractor`. DI swap in `Program.cs`. Existing `NoOpPdfTextExtractor` stays for tests.
2. Optional Azure Document Intelligence analyzer (`IPdfDocumentAnalyzer`) behind `PdfExtractionOptions.AzureDocIntelEndpoint` config — skipped (returns empty) when unconfigured. New service writes a sibling `OcrUsageRecord` (or a structured log line) for cost telemetry; does NOT use `AiUsageRecord`.
3. `AiCredentialResolver.PlatformOnlyFeatures` includes `AdminListeningDraft`, `AdminPronunciationDraft`, `AdminVocabularyDraft`, `AdminConversationDraft`. Backfilled in `docs/AI-USAGE-POLICY.md` §5.
4. Player auto-seeks `audio.currentTime` to the first un-completed extract's `audioStartMs / 1000` on section enter. Pause / advance soft-fired when `audio.currentTime * 1000 >= audioEndMs` (only when `modePolicy.canScrub` is false → exam mode). Manual scrub still works in practice mode.
5. `getListeningSession` failure path handles `content_locked` and renders the same `ContentLockedNotice` instead of generic error.
6. `ListeningSampleSeeder` runs at startup behind a config flag (`Seed:ListeningSamples=true`). Idempotent (keyed on `MediaAsset.Sha256` + `ContentPaper.Slug`). Produces 3 draft papers `listening-sample-1|2|3-medicine`. Admins still must click Publish.
7. New backend tests: `PdfPigExtractorSmokeTests`, `AiCredentialResolverPlatformOnlyTests` (regression for the gap), `ListeningSampleSeederTests`.
8. New frontend tests: `ListeningStructureEditor.test.tsx` (smoke), `app/listening/player/[id]/__tests__/cue-point-seek.test.tsx`.

## 7. Risk register

- **PdfPig license / native deps:** PdfPig is MIT, pure C#, no native deps — safe.
- **Azure DocIntel cost classification:** Not an LLM, must NOT route through `AiGatewayService`. Per research: emit a separate `OcrUsageRecord` instead, or just structured log for now.
- **MP3 size:** 33–53 MB ≤ `StorageOptions.MaxAudioBytes = 150 MiB` — safe.
- **Seeder duplicates:** SHA-256 idempotency dedupes; `ContentPaper.Slug` unique guard prevents double-create.
- **Sample 2 OCR optional in dev:** Without Azure config, AI extraction will produce a partial draft; admin completes Answer Key by hand. Acceptable.
