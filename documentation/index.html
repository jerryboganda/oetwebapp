Installation

Follow the steps below to set up your site template:

  1. Unzip the downloaded package and open the Axelit-next folder to find all the files. You will need to upload these files to your hosting web server using HTTP or localhost in order to use it on your website.
  2. Below is the folder structure and needs to be uploaded to your website or localhost root directory:
    • Axelit-next/src - Contains all of the pages referenced
      • Axelit-next/src/scss - Stylesheet files
      • Axelit-next/public/images - Image files
      • Axelit-next/src/app.tsx - Base folder for routing configurations and related files.
      • Axelit-next/src/data - Dedicated folder for storing chart data and other application-specific data requirements.
      • Axelit-next/src/app - Grourp Routing.
    • Axelit-next/src/app/page.js - Homepage
  3. You should upload all or specific JS - JSX files as per your need.
  4. You are good to go for adding your content now!
  5. To run the project, you will need Node.js and npm installed on your system.
  6. If you want to compile a template , use npm run dev
  7. Use the following command to build project : npm run build

Group Routing in Next.js App Router

Step 1: Create the App Router Structure

Make sure you’re using the app/ directory-based routing system. Your project should have this structure:

/src
└── /app

Note:If you're still using pages/, migrate to the App Router system first.

Step 2: Create a Group Folder

Create a folder inside app/ and wrap the folder name in parentheses.

app/
└── (group-name)/

Note:Group folders are only for organizing code — they do not become part of the route URL.

Step 3: Add Routes Inside the Group Folder

Inside your group folder, create subfolders or pages as usual. You can also add shared components, layouts, loading states, and error boundaries inside this group.

app/
└── (animation)/
         ├── _components/
         │      └── AnimateUse.tsx
         └── animation/
                 └── page.tsx

If you're still using pages/, migrate to the App Router system first.

Dark/Light Mode

The HeaderMode component provides a simple way to toggle between light and dark themes in a Next.js


import React from "react";
import { useEffect, useState } from "react";
import { MoonStars, SunDim } from "@phosphor-icons/react";

const HeaderMode = () => {
  const [theme, setTheme] = useState("light");

  useEffect(() => {
    const storedTheme = localStorage.getItem("theme-mode");
    if (storedTheme) {
      setTheme(storedTheme);
      document.body.classList.add(storedTheme);
    } else {
      document.body.classList.add("light");
      setTheme("light");
    }
  }, []);

  const toggleTheme = () => {
    const newTheme = theme === "light" ? "dark" : "light";
    setTheme(newTheme);
    document.body.classList.remove("light", "dark");
    document.body.classList.add(newTheme);
    localStorage.setItem("theme-mode", newTheme);
  };

  return (
    <div className="header-dark" onClick={toggleTheme}>
      <div className={`sun-logo head-icon ${theme === "dark" ? "sun" : ""}`}>
        <MoonStars size={24} />
      </div>
      <div
        className={`moon-logo head-icon ${theme === "dark" ? "moon" : ""}`}
      >
        <SunDim size={24} />
      </div>
    </div>
  );
};

export default HeaderMode;

Ltr Layout

For enabling the Ltr Layout on your website, simply apply the ltrclass to the body tag and add the dir="ltr" attribute to your HTML tag

                                  
    
       <html lang="en" dir="ltr">
         <body className="ltr">
                
         </body>
       </html>

Rtl Layout

For enabling the Rtl Layout on your website, simply apply the rtlclass to the body tag and add the dir="rtl" attribute to your HTML tag

                                  
   
       <html lang="en" dir="rtl">
       <body className="rtl">
            
       </body>
     </html>

Box Layout

For enabling the Box Layout on your website, simply apply the box class to the body tag
